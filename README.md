# OTO — オフライン音楽PWA

MP3をブラウザ内のオフラインライブラリへ保存し、スマホで通信なしでも再生できるWebアプリです。

## できること

- 手元のMP3を複数選んで端末内へ保存
- 権利を持つYouTube動画をMP3へ変換して、そのまま端末内へ保存
- オフラインでライブラリを表示・再生
- プレイリストの作成・まとめて再生・シャッフル再生
- 保存した曲のカット編集（波形を見ながら複数範囲を削除、カット後をプレビュー再生してMP3で保存）
- Apple Music風のライブラリ画面（MP3ダウンロード / プレイリスト / 曲）からタップで移動
- 別タブや別アプリへの移動中も標準の音声再生を継続
- ロック画面や通知領域の再生・停止・前後曲操作（対応ブラウザ）
- ホーム画面へ追加してアプリ風に起動

曲データはIndexedDB、画面一式はService Workerへ保存します。外部ストレージへ曲をアップロードしません。

## PCで起動

Windowsでは `起動.bat` をダブルクリックします。初回だけ変換用ツールを準備するため、少し時間がかかります。

PowerShellから起動する場合:

```powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
```

起動後は `http://127.0.0.1:8765` を開きます。

## iPhone単体で使うために

変換サーバーをRenderへ公開すると、PCを起動していなくてもiPhoneだけで利用できます。Docker構成と `render.yaml` は準備済みです。

1. このフォルダを自分のGitHubリポジトリへpushします。
2. [Render](https://render.com/) でGitHubアカウントを接続し、Blueprintとしてこのリポジトリを選びます。
3. `AUDIO_TOOL_ACCESS_CODE` を尋ねられたら、好きな6桁の数字を設定します。
4. デプロイ完了後に発行される `https://...onrender.com` をiPhoneのSafariで開きます。
5. Safariの共有メニューから「ホーム画面に追加」を選びます。

無料インスタンスは15分使わないと休止するため、最初に開いたときだけ起動に1分ほどかかることがあります。MP3本体はRenderへ永続保存せず、iPhoneへ転送後にサーバーから削除します。iPhone側ではブラウザの専用領域へ保存されます。

- iPhone / iPad: Safariの共有メニュー →「ホーム画面に追加」
- Android: Chromeのメニュー →「アプリをインストール」または「ホーム画面に追加」

OSがブラウザを強制終了した場合や省電力設定によっては、バックグラウンド再生が止まることがあります。

## テスト

```powershell
py -m unittest -v
```

## 注意

自分で作成した動画、または変換・保存・利用の許可を得た動画だけに使用してください。プレイリストの一括取得には対応していません。
